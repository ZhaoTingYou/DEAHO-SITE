## 1. 设计系统

### 1.1 色彩(亮调铁律)

```css
:root{
  --bg:       #F8F6F2; /* 暖象牙白 · 整站主基调,占比最大 */
  --white:    #FFFFFF; /* 纯白 · 产品"聚光"段,与象牙白形成细腻层次 */
  --primary:  #101D30; /* 深海军蓝 · 当"墨色"用:标题/正文/细线/页脚/极少量小色块 */
  --accent:   #7A2230; /* 酒红 · 仅点睛 */
  --text:     #1A1A1A; /* 近黑正文(纯文字,绝不当背景) */
  --subtext:  #666666; /* 次级文字 / 说明 */
  --hairline: rgba(16,29,48,.14); /* 海军蓝细分隔线 */
  --on-navy:  #F8F6F2; /* 万一用到海军蓝小色块/页脚时的文字色 */
}
```

**铁律**
- 整站**亮调为主**:默认底 `--bg`(暖象牙白),产品聚光段用 `--white`。**不使用黑色 / 近黑背景。**
- 层次来自 **象牙白 ↔ 纯白 的细腻切换 + 柔和长投影 + 大图**,**不靠明暗反色**。
- 海军蓝 `--primary` 主要当"**墨色**":大标题、正文、细线、页脚。**不大面积铺作背景。** 若全站需要一处"重锚"(通常仅页脚,或某一句收束),可用**小面积**海军蓝块、文字反白 `--on-navy`——但要少而精,整站仍是亮调。
- 酒红是手术刀:当前态下划线、单个强调词、COMING SOON、细分隔。**绝不大面积。**
- 文字优先用 `--text`/`--primary`;次要信息 `--subtext`(小字 ≥14px)。

### 1.3 间距与栅格
- 容器最大宽 1440px,内边距 24→48→80px。段落竖向节奏 `clamp(96px,12vw,180px)`(手机 64–80px)。**留白是亮调奢华的灵魂,绝不挤。**
- 12 栏栅格,24px 间隔;多处刻意破栅格(不对称、叠压)。
- 断点:手机 <768、平板 768–1024、桌面 >1024、宽屏 >1440。
- hero 满幅(100vw),产品格在容器内。尺寸/留白参照 Omega / Montblanc / Porsche。

### 1.4 全局动效语言

```css
:root{
  --ease:        cubic-bezier(.22,.61,.36,1);
  --ease-expo:   cubic-bezier(.16,1,.3,1);
  --dur-hover:   600ms;
  --dur-reveal:  900ms;
  --dur-hero:    1200ms;
}
```

四个**全站统一动作**:
1. **Hover Zoom(品牌招牌交互,源自你的基调表)。** 图片/卡片外层 `overflow:hidden`,hover 时内层图 `scale(1.04)`,`--dur-hover` `--ease`。文字链:酒红下划线从左到右扫入。
2. **滚动揭示。** 元素从 `translateY(28px)`+`opacity:0` 落定到 `0/1`,`--dur-reveal` `--ease-expo`,进入视口约 15% 触发;同组子元素错峰 80–120ms。
3. **亮调层次切换。** 段落间在 `--bg`(象牙)与 `--white`(纯白)之间细微切换,配合**柔和投影渐显**做出"聚光 / 退场"层次——**不做明暗反色,不出现深色背景**。
4. **入场编排。** 每页只做一次有编排的入场(见各页"招牌动作"),其余靠滚动揭示。**绝不满屏碎动效。**
- **`prefers-reduced-motion`:** 关闭漂浮 / 视差 / zoom,只保留瞬时淡入。
- 实现提示:平滑滚动 Lenis;揭示 Framer `whileInView`;滚动驱动 `useScroll/useTransform`;count-up 轻量 hook;SVG 描线 `pathLength`+滚动进度。详见《动画实现要点》。

# 1.6 各页版式差异化总纲(关键:一眼认出是哪一页)

> **目标:** 不靠图和文字,**单看版式骨架就知道这是哪一页**。每页钉死一套**专属布局原型 + 专属网格 + 一个"标识性结构元素"(signature element)**。
> **红线:这套结构元素是该页专属,绝不挪用到别页。** 六页的"骨架"必须明显不同;只有"设计系统"(色/字/间距/动效语言)是统一的。

### 各页"版式身份证"

| 页面 | 布局原型(骨架) | 专属网格/排布 | 标识性结构元素(只此一页) | 信息密度 / 节奏 |
|---|---|---|---|---|
| **HOME** | **满幅电影序列**:整屏 hero 视频 → 横向产品陈列 → 数字带 → 四篇章入口 | 段段满幅、居中对齐为主 | **背景视频 hero + 四篇章"门户卡"** | 低密度、慢、仪式感 |
| **CHRONICLE** | **垂直时间轴**:一条贯穿全页的竖轴,内容**左右交错**挂在轴两侧 | 非对称双栏(轴居中偏左),年份做锚点 | **会自我绘制的时间轴竖线 + 年份锚点 + FIRST 徽章** | 中密度、线性、археологический(考据感) |
| **LEGACY** | **居中宣言 + 满屏数字**:大量留白,内容**水平居中堆叠**,几乎不分栏 | 单列居中,超大留白,数字独占整屏 | **逐行光扫的哲学句 + 整屏 `<StatRow>` 巨数字 + 合作墙阵列** | 极低密度、最慢、最庄重 |
| **SPECIALTY** | **钉屏分步 / 横向工序**:左固定标题轴,右内容随滚动推进 | **左右分栏**:左=步骤号/标签(粘住),右=微距大图 | **钉屏七步 `01→07` + 半透明巨步骤号 + 微距三连** | 高密度、强推进感、工业 |
| **NEWS** | **杂志网格**:卡片化、可筛选,信息块并置 | **多列瀑布/网格 + 顶部筛选 chips** | **分类 chips + 日期/标签元信息 + COMING SOON 卡** | 最高密度、最轻快、可扫读 |
| **GOLF** | **产品配置页**:hero → 选项区 → 规格/刻字 → 实物 | **不对称叠图 + 选项卡组**(配置器布局) | **杆头/杆色选择器 + 刻字规格示意线 + 叠压实拍** | 中密度、互动、玩味 |

### 落地约束(给 Codex)
1. **对齐方式各页不同**:HOME 居中、CHRONICLE 左右交错、LEGACY 纯居中、SPECIALTY 左粘右滚的分栏、NEWS 网格左对齐、GOLF 不对称叠压。**不要六页都用同一种"居中标题+两栏卡片"**。
2. **首屏(hero)结构各页不同**,一眼区分:HOME=满幅视频;CHRONICLE=轴线起点 + `EST.1988`;LEGACY=居中哲学句(无图占大半);SPECIALTY=微距特写铺满;NEWS=明快通栏 + 即时感;GOLF=悬浮产品 + eyebrow。
3. **`SectionIntro` 的呈现按页变体**:CHRONICLE 的标题挂在轴上;LEGACY 的标题居中且字超大、留白超大;NEWS 的标题旁直接跟筛选 chips;SPECIALTY 的标题是粘住的左轴。**同一个组件,不同 variant,不要长一样。**
4. **节奏/密度按上表**:LEGACY 段间距最大(可达 `--space` 上限的 1.3×)、NEWS 最紧凑(卡片网格、段间距收到下限)。靠"疏密"也能一眼分辨。
5. 标识性结构元素**互不出现在别页**(例:时间轴只在 CHRONICLE;钉屏七步只在 SPECIALTY;配置器只在 GOLF;筛选 chips 只在 NEWS/collection)。

> 一句话:**统一的是"语言"(色/字/动效),不同的是"句式"(每页的骨架与对齐)。** 六页放在一起翻,应像同一本书里六种排版完全不同的章节。

# 1.7 各页核心视觉母题(决定"一眼入魂")

> §1.6 管"骨架不同";本节管**"气质一眼可辨"**——给每页一个**专属视觉母题(signature motif)**:一套别页绝不出现的视觉语汇(排印手法 + 标志元件 + 质感 + 主导画面)。
> **检验标准(贯穿全程):把任一页截一张图、遮住所有文字内容,旁人也能立刻说出"这是 XX 板块"。** 做不到 = 没达标。
> 母题之间**互斥**:一页的母题元件绝不出现在另一页。

### 各页"视觉母题卡"

**HOME —— 母题《幕启 · 影院银幕》**
- 主导画面:**整屏循环视频**当幕布;一切从这块"银幕"展开。
- 排印:超大 Cormorant 衬线居中压在影像上,字少而重,像电影片名。
- 标志元件:底部细长**下滑指示**、四篇章的**满幅"门户卡"**(整块图 + 一行大标题,点击进入)。
- 一眼感:像走进一家旗舰店/电影开场——**沉浸、仪式、留白最大**。

**CHRONICLE —— 母题《档案 · 时间轴》**
- 主导画面:一条**贯穿到底的竖线**,内容像档案卡片左右挂在轴上。
- 排印:**超大年份**(1988…)做锚点,衬线+等宽数字混排,旁边小字注解,像史料条目;**FIRST 红章**。
- 标志元件:轴线、年份锚点、红色 FIRST 徽章、"老照片冲洗"显影质感。
- 一眼感:像在读一份**带年份刻度的编年档案**——线性、考据、有重量。

**LEGACY —— 母题《碑文 · 巨数字》**
- 主导画面:大片留白里,**居中的一句话**和**独占整屏的巨大数字**(38 / 200,000+)。
- 排印:哲学句用衬线、**字号极大、行距极松**,像碑文/题词;数字大到近乎海报。
- 标志元件:逐行光扫的句子、整屏 StatRow、安静的合作墙阵列。
- 一眼感:像走进一间**只挂着一句箴言的纪念厅**——克制、庄重、最慢。

**SPECIALTY —— 母题《工序 · 解剖图》**
- 主导画面:**极致微距 + 左侧粘住的工序轴**;画面像放大镜下的金属解剖。
- 排印:**半透明巨步骤号 `01–07`** 压在微距图侧,等宽英文小标签(DESIGN SKETCH…),技术蓝图味。
- 标志元件:钉屏推进的七步、巨步骤号、微距三连、细节标注线。
- 一眼感:像在看一份**工艺白皮书/技术拆解**——精密、推进、信息密。

**NEWS —— 母题《刊物 · 报头网格》**
- 主导画面:**杂志/报纸式的卡片网格**,信息块并置、可扫读。
- 排印:**报头式大字 `NEWS` + 一条细规则线(rule line)** 横贯页顶;每张卡带**分类标签 + 日期**(像刊期/栏目);可选一条**滚动快讯条(ticker)**。
- 标志元件:顶部筛选 chips、日期/栏目元信息、COMING SOON 卡、ticker。
- 一眼感:翻到这页**立刻"这是新闻/刊物版"**——最轻快、最密、最当下。

**GOLF —— 母题《配置 · 产品台》**
- 主导画面:**悬浮的产品 + 选项面板**,像可定制的产品配置台。
- 排印:产品名/材质用克制无衬线,旁边**规格示意细线 + 刻字预览**,工业产品册味。
- 标志元件:杆头/杆色**选择器卡组**、刻字规格线、不对称叠压实拍。
- 一眼感:像进入一个**可上手把玩的定制台**——互动、玩味、聚焦单品。

### 落地约束(给 Codex)
1. **每页至少落实 2 个该页专属母题元件**(见各卡"标志元件"),且这些元件**不得在别页出现**。
2. **排印手法按页区分**:HOME=电影片名式居中巨衬线;CHRONICLE=年份锚点+档案注解;LEGACY=碑文式超大行距;SPECIALTY=半透明巨编号+等宽标签;NEWS=报头大字+规则线+日期标签;GOLF=产品名+规格细线。
3. **NEWS 必须有"刊物感"硬元件**:页顶报头 `NEWS` + 规则线、每卡分类/日期、(建议)一条 ticker——确保"一眼是新闻"。
4. 这些母题与 §1.6 骨架、§2 招牌动画**叠加生效**;母题负责"气质一眼可辨",骨架负责"结构不同",动画负责"记忆点"。

> 记忆口诀:**HOME 银幕 · CHRONICLE 时间轴 · LEGACY 碑文 · SPECIALTY 解剖图 · NEWS 报头 · GOLF 配置台。** 六个母题互不串门。

- **第一行(utility bar,高约 48–56px):**
  - **左:logo `DAEHO`** —— Cormorant Garamond,宽字距(`D A E H O`),**靠左,不居中**。点击回首页。
  - **右:语言切换 `KO / EN`** + 分隔 + **三个外链 `대호 · OH · VULCAN`** + 分隔 + **`CONTACT · 문의` 入口(常驻最右)**。语言:当前态 `--text`/反白,非当前态 `--subtext`;外链:Inter Medium 大写小字,`href` 暂空(`#`)、`aria-disabled`、透明度 .7、无 hover 下划线、`config.externalLinks` 配置;**CONTACT:全站常驻、最显眼的联系入口**——比外链更突出,用**酒红 `--accent` 描边小药丸(pill)或酒红文字**,链到 `/contact`,hover 轻微填色/反白;它属于"工具区",**不参与 mega menu、不进主导航那条品牌路径**。透明态下 CONTACT 用反白描边,象牙态下用酒红。
- **第二行(主导航,高约 56–60px):**
  - `HOME · CHRONICLE · LEGACY · SPECIALTY · NEWS · GOLF`,Inter Medium 大写,字距 0.12em。**居中或靠左对齐**(建议靠左,与 logo 左对齐成轴线,更有秩序;若想更经典也可居中——二选一,全站统一)。
  - 当前页:酒红 `--accent` 下划线。
  - LEGACY / SPECIALTY 带子级 → hover 出 **mega menu**(见下)。其余无子级。
- 两行之间一条极淡 hairline 分隔(透明态下可省略,象牙态下显示)。

### Mega Menu(LEGACY / SPECIALTY 的 hover 下拉)

- **形态:通栏大菜单**——面板**横跨整个顶栏宽度**(满宽),从主导航底缘向下滑出,象牙/纯白底 + 柔和长投影 + 顶部一条酒红细线。
- **内容:每个子项 = 标题 + 一行小字描述**(利用 mega menu 的空间):
  - **LEGACY**
    - `LOYALTY` — 신의 · 오래도록 함께한 신뢰(长期合作与客户维持)
    - `CREDIBILITY` — 신뢰 · 숫자로 증명되는 믿음(信赖数据与合作伙伴)
    - `ACHIEVEMENT` — 성취 · 기록으로 남은 인정(认可与话题项目)
  - **SPECIALTY**
    - `TECHNIQUE` — 공정 · 7단계의 정성(7 步制作流程)
    - `COLLECTION` — 작품 · 완성된 결과물(作品集画廊)
  - 可在面板右侧放一张**小预览图**(对应该板块的母题缩略,如 LEGACY 的巨数字、SPECIALTY 的微距),强化"一眼是哪块"。无图时留白即可。
- **触发与容错:** hover 该主项 **或** 面板本身都保持打开;移出后**延迟 ~150ms 再关**(给斜向移动到面板的余地,避免一移就消失)。点击主项本身 = 进该板块索引页。
- **键盘/无障碍:** 主项可聚焦,`aria-haspopup`/`aria-expanded`;`Esc` 关闭;子项 Tab 可达。**触摸设备无 hover** → 点击主项展开面板(再点进子项),或直接进索引页(移动端走抽屉,见 §4.2)。

### 配色 / 状态

- **透明态(覆盖在 hero 视频/大图上,初始):** 背景透明,**文字反白 `--on-navy`/象牙白**;底部无边框。为保证反白可读,hero 顶部叠极淡暖白渐变(见 §2.1)。
- **象牙态(滚动后):** 背景 `--bg` 象牙白(可加轻微毛玻璃 `backdrop-blur` + 细 hairline 底边),**文字转海军蓝 `--primary`**;mega menu 面板恒为象牙/纯白底。
- 两态之间所有颜色过渡 300ms。

### 滚动行为(方向感知:下滚隐藏 / 上滚出现)

- **下滚** → 顶栏整体向上滑出隐藏(`translateY(-100%)`);**上滚** → 立即滑回出现。
- 在页面**最顶部时始终显示且为透明态**;一旦离开顶部并进入"象牙态"。
- 隐藏时若 mega menu 开着 → 一并关闭。
- 细节见《动画要点》§Header。

### 动画(精细化清单)

1. **页面载入:** utility bar 先淡入(logo 从 `−8px` 落下),主导航项**逐个错峰**淡入(每项 60–80ms),当前页下划线最后"画出"(scaleX 0→1)。
2. **链接 hover:** 文字色微亮 + **酒红下划线从左到右扫入**(`transform-origin:left; scaleX 0→1`,250ms);移出反向收回。
3. **Mega menu 展开:** 面板高度/透明度从 0 展开(`--ease-expo`,360ms),内部子项**错峰上浮**(每项 40–60ms);顶部酒红线先于内容 80ms 画出。关闭 200ms 收起。
4. **透明↔象牙 切换:** 背景色 + 文字色 + 底边 hairline 同步过渡 300ms;毛玻璃淡入。
5. **方向隐藏/出现:** `translateY` 用 `--ease`,260ms;隐藏与出现速度可略不同(出现更快,~200ms,更跟手)。
6. `prefers-reduced-motion`:取消滑动/错峰,改为瞬时显隐与即时下划线。

> 一句话:**两层、透明起步、上滚即现、hover 出通栏大菜单**——精致但不喧哗。

> 每页:**设计概念 → 视觉设计(逐段)→ 动画设计(入场/滚动/Hover/招牌动作)。** 所有背景只在象牙白/纯白间走,无深色场。

## 2.1 HOME · 首页

> **设计概念:《晨光 / First Light》。** 在明亮通透的展厅光里,一枚戒指被柔光托起——一口气讲完品牌论证,再送进四大篇章。

### 视觉设计
- **S1 Hero(满幅背景视频,高调)。** 参照 Omega / Porsche 首页:一支**静音、自动、循环、内联**的背景影片铺满首屏,文字压在其上。eyebrow `KOREA'S CHAMPIONSHIP RING AUTHORITY`,大字 `38 YEARS OF CHAMPIONS`(海军蓝或反白,见下),韩文副标 `승리의 순간을, 영원의 형태로.`,底部下滑提示。
  - **视频规格(见下方"Hero 视频规范")。** 内容:明亮高调的戒指特写/工坊柔光镜头,缓慢运镜,无人脸、无文字;基调与全站一致(象牙/暖光,不阴郁)。
  - **现在没有视频 → 用静态图占位:** 先用 `home_hero.png` 作 **poster 海报 + 占位**;`<HeroMedia>` 组件做成"有视频放视频、无视频回退到 poster 图"。日后把视频文件丢进 `/public/videos/home_hero.mp4`(+ `.webm`)即自动启用,**无需改版式**。
  - **文字可读性:** 视频/图上压一层极淡的**暖白渐变**(从底部或左侧,象牙色 0→约 35% 透明),保证海军蓝或反白文字清晰;**不使用黑色蒙版**(守亮调)。文字颜色二选一并全站统一:亮底用海军蓝 `--primary`;若局部画面偏亮金属、对比不足,则该处文字用反白 `--on-navy` + 更强暖白渐变。
- **S2 代表作(纯白聚光段)。** eyebrow `SIGNATURE WORKS` / `대표작`;3–5 枚戒指横向陈列(`home_ring_01..05`,1:1),每卡一行韩文说明,点进 → `/specialty/collection`。
- **S3 数字(象牙白)。** 三组**海军蓝大数字**:`1988 설립` · `38 년의 업력` · `200,000+ 제작된 반지`;底可叠极淡暖金属光纹理 `home_stats_bg.png`(低不透明度,仍保持亮)。
- **S4 四大篇章(纯白/象牙交替)。** 四张大编辑卡(亮图 + 标题,标题用海军蓝压在图较亮处或图下):
  CHRONICLE `우리가 시장을 만들었다` · LEGACY `그래서 신뢰할 수 있다` · SPECIALTY `그래서 우리가 최고다` · NEWS `그리고 지금도 만들고 있다`(`home_pillar_*`,3:4)。
- **S5 GOLF 预告(纯白满幅,高调)。** 大字 `FORM OF THE GAME` + `골프의 구조를 하나의 오브젝트로.`,复用 `golf_hero.png`,CTA → `/golf`。

### 动画设计
- **入场(招牌动作):** 导航淡降(400ms)→ hero 大字**逐词上浮**(每词 `translateY(40→0)+opacity`,错峰 120ms,`--ease-expo`)。**背景视频在首帧 = poster 海报图**,可解码后做 `scale(1.04→1.0)` 的极缓推近(2s)无缝过渡到播放,避免"图突然变视频"的跳变;若回退为静态图,则图本身做该推近 + 极缓漂浮。底部下滑提示呼吸。
- **滚动:** S3 数字 count-up(2s,千分位);S4 四卡错峰揭示;象牙↔纯白细微切换。**Hero 视频随滚动离场时:轻微视差(比内容慢 ~20%)+ 略微缩放,但滚出首屏后应暂停播放省电(见规范)。**
- **Hover:** 代表作/篇章卡 zoom 1.04,投影同时略加深(更"浮起")。
- **轻视差:** hero 媒体随滚动比背景略慢上移。

### Hero 视频规范(技术硬要求 · 给 Codex)
> 目标 = Omega / Porsche 那种"进站即播的全屏背景影片",但**自动播放有硬性条件,缺一不可**。做成可复用组件 `<HeroMedia poster videoSrc?>`。

- **必备属性(缺任一则手机不自动播):** `autoPlay`、`muted`(且 React 里用 `muted` 属性 + DOM `video.muted=true` 双保险)、`loop`、`playsInline`(iOS 必需)、`preload="metadata"`、`poster={home_hero}`。
- **回退链:** 有 `videoSrc` → 放视频(poster 作首帧/加载占位);**无 `videoSrc` → 直接渲染 `home_hero.png`**(当前阶段就是这个)。组件对外接口不变,日后只需传入 `videoSrc`。
- **多格式:** 同时给 `<source>` `.webm`(VP9/AV1,体积小)与 `.mp4`(H.264,兼容)。
- **尺寸/裁切:** `object-fit:cover` 满幅;约 16:9 或更宽;**桌面 ≤ 6–10MB、≤ 约 20s 循环**(品牌站够用,别更大)。
- **省电/性能:** 用 IntersectionObserver,Hero **滚出视口即 `pause()`、回到视口 `play()`**;`prefers-reduced-motion: reduce` 时**不自动播**,只显 poster 图(可给一个播放按钮让用户手动播)。移动端可选:用更短/更小码率的竖版或同一支 + 更强压缩(见 §4 移动端)。
- **可读性蒙版:** 视频上叠**暖白渐变**(非黑),保证文字 AA 对比;字色按上文 S1 规则。
- **无障碍:** 背景视频是装饰性 → `aria-hidden` + 无音轨;关键信息全部在叠加的文字里(不依赖视频内容)。
- **资源位:** `public/videos/home_hero.mp4`、`home_hero.webm`;poster 复用 `public/images/home_hero.png`。

## 2.2 CHRONICLE · 沿革

- **S2 时间轴(象牙白)。** 一条**海军蓝细竖线**为轴(桌面居左偏中),年份用 `stat` 海军蓝刻度;条目桌面左右交替、手机堆叠。6–8 个占位里程碑,"国内首个/首次引入/专利"挂 **FIRST 徽章(酒红)**。图 `chronicle_milestone_01..06`(3:2)。

### 动画设计
- **招牌动作 —— 时间轴"自我绘制":** 海军蓝竖线是 SVG,`pathLength` 绑滚动进度**随下滑一笔笔画下来**;经过节点时小圆点点亮(scale 0→1 + 酒红描边一闪)。

- **左右交替进场:** 左条目从 `−40px`、右条目从 `+40px` 滑入 + 淡入。
- **FIRST 徽章:** 进入时轻弹(scale 0.8→1.05→1)。

## 2.5 NEWS · 动态

> **设计概念:《当下的脉搏 / The Current Pulse》。** 全站最轻快、最"杂志感"的一页(也最亮),证明"现在仍在持续创造"。

### 视觉设计
> **母题《刊物 · 报头网格》(§1.7):一眼就是"新闻/刊物版"。** 以下报头、规则线、日期/栏目标签、ticker 为本页专属,别页不得出现。
- **S0 报头(Masthead,页顶通栏)。** 像报纸刊头:左侧**超大 `NEWS`**(海军蓝衬线),右侧一行小字 `DEAHO JOURNAL · 발행 2026`(刊期感),下方一条**贯通的细规则线(rule line)**。规则线之下可选一条**滚动快讯条 ticker**(等宽小字横向滚动:最新动态/标题轮播,海军蓝、低调)。这一组合是"新闻感"的核心。
- **S1 Hero(纯白,通透明快)。** 紧贴报头之下:大字 `지금, 대호` + 副标;图 `news_hero`。比其他页 hero **更矮、更横向**(像头版头条 banner,不占满屏),强化"刊物"而非"展厅"。
- **S2 Featured / 头条(象牙白)。** 一张大 `MediaText` 作"头版头条",eyebrow `COMING SOON`(酒红)或 `TOP STORY`;图 `news_featured` 用**象牙白半透柔雾**蒙一层(高调"半隐"),不是暗场。配**栏目标签 + 日期**。
- **S3 新闻网格。** 报刊式卡片网格(比别页更密、左对齐),每卡:缩略图 + **栏目标签(像报纸版块名)+ 日期** + 韩文标题,6 张 `news_card_01..06`(3:2)。顶部**筛选 chips** = 版块切换。栏目:최근 제작 사례 · 프로젝트 스토리 · 언론/피처 · 협업 · 우승반지/귀금속 지식。

### 动画设计
- **招牌动作 —— ticker 流动 + COMING SOON 微光:** 页顶 ticker 横向匀速滚动(可 hover 暂停);`COMING SOON` 标签一道细高光周期扫过(2.5s);featured 柔雾在 hover **轻微淡开**(亮场"揭幕")。
- **新闻格:** 卡片错峰揭示;hover **抬升 + zoom**(`translateY(-6px)` + 图 1.04 + 投影加深),比其他页略跳。
- **分类筛选:** chips FLIP 重排;当前 chip 酒红下划线。
- **报头规则线:** 入场时从中间向两端"画开"(scaleX 0→1),呼应刊物感。
- 元信息(栏目/日期)比标题晚 150ms 淡入。

# 2.7 详情页与子页(列表点进去之后)

> 原则:**详情页要继承所属板块的母题气质,而不是退化成千篇一律的文章模板。** 即"从 NEWS 点进去,详情页仍像刊物内页;从 COLLECTION 点进去,详情页仍像珠宝陈列"。统一的是版心、字体、动效;不同的是骨架与母题元件。
> 路由约定:`/news/[slug]`、`/specialty/collection/[slug]`、`/chronicle/[year]`(或锚点)、`/golf/inquiry`、`/contact`。

## 2.7.1 NEWS 文章详情页 · `/news/[slug]`
> **母题延续《刊物 · 报头》:像翻到杂志内页。** 单栏窄版心(阅读舒适)+ 刊物式版头。

**视觉设计**
- **S0 文章版头(沿用报头气质)。** 顶部一条细规则线;**栏目标签(酒红)+ 日期** 在最上;其下**大标题**(韩文 MaruBuri / 英文 Cormorant),副标/导语用 `--subtext`;作者/来源等元信息一行小字。
- **S1 主视觉。** 一张满版心宽图(`news_detail_hero` 占位,16:9)。
- **S2 正文。** **单栏窄版心(最大 ~720px 居中)**,正文 Pretendard/Inter 18–20px、行高 1.7;支持小标题、引用块(左侧酒红竖线)、图文段、双栏图。段落之间留白充足。
- **S3 文末。** 标签 chips + "분享/공유"按钮 + **CTA:`이 프로젝트가 궁금하신가요? → 문의하기`**(进 `/contact` 或弹窗,见 §2.7.5)。
- **S4 相关报道。** "다음 소식" 3 张卡(复用 NEWS 网格卡),回流列表。

**动画**
- 版头规则线入场"画开";标题逐行上浮;主图 `scale 1.03→1.0`。
- 正文滚动揭示(段落错峰);引用块左侧酒红竖线随进入"画下"。
- 阅读进度:顶栏下方一条**极细酒红进度条**(随滚动,可选)。
- 相关卡错峰 + hover 抬升 zoom。

## 2.7.2 COLLECTION 作品详情页 · `/specialty/collection/[slug]`
> **母题延续《工序 · 解剖》+ 珠宝陈列:像高级珠宝产品页。** 左图廊 / 右信息的经典两栏,微距特写为主角。

**视觉设计**
- **S1 作品主区(两栏)。** 左:**大图 + 缩略图廊**(主图 + 3–5 张细节微距,可切换;`collection_detail_01..n` 占位,1:1/4:5);右:**作品信息面板**——作品名(衬线大字)、类型(冠军/任官/订制)、**规格清单**(材质 금/은、宝石、年份、为谁制作等,用细线分隔的键值表)、一段作品故事。右栏在桌面**粘住(sticky)**,左图随滚动。
- **S2 细节微距条。** 横向 2–3 张极致微距(刻字/雕刻/镶嵌),延续 SPECIALTY 的"解剖"气质。
- **S3 工艺呼应。** 一句"본 작품에 적용된 공정" + 小链接回 `/specialty/technique`(把单品和工艺叙事连起来)。
- **S4 CTA:** `이와 같은 작품을 의뢰하고 싶다면 → 맞춤 제작 문의`(进 `/contact`,预填"订制咨询")。
- **S5 相关作品。** 同系列/同类型 3–4 枚(复用 collection 卡)。

**动画**
- 进场主图从缩略图"放大就位"(shared layout);切换缩略图 = 主图交叉溶解。
- 右信息面板 sticky;规格表逐行(键值)错峰淡入,细线"画出"。
- 微距条 hover zoom;`scale 1.1→1.0` 进入。

## 2.7.3 CHRONICLE 里程碑详情 · `/chronicle/[year]`
> **母题延续《档案 · 时间轴》:像翻开一页档案卷宗。** 不做成普通文章,保留年份锚点与档案感。

**视觉设计**
- **S1 卷宗页头。** 超大**年份**(stat 巨字,海军蓝)+ 标题 + 若是"国内最初"则挂 **FIRST 红章**;一行"이 해의 의미" 注解。
- **S2 档案主体。** 左侧一条短轴线/索引(呼应时间轴母题),右侧:这一里程碑的故事 + 当年影像(`chronicle_detail_01..n` 占位,3:2),可含"기술/특허 설명""관련 프로젝트"分块。
- **S3 上下文导航。** **← 이전 연도 / 다음 연도 →**(在编年里前后翻,强化"档案"体验),回时间轴的链接。
- **S4 CTA(轻):** `대호의 기술이 궁금하다면 → SPECIALTY` / 或 `문의하기`。

**动画**
- 年份巨字 blur→focus 进入;短轴线描出;影像"老照片显影"(降饱和→正常)。
- 前/后年导航 hover 时年份数字微移。

> **可选简化:** 若里程碑内容不多,CHRONICLE 也可**不做独立详情页**,改为时间轴上点击**就地展开**(accordion / 弹层)。两种都给 Codex:默认做就地展开,内容够多再升级为独立 `/chronicle/[year]`。

## 2.7.4 GOLF 订购/咨询流程 · `/golf/inquiry`

**视觉设计**

# 4. 移动端设计

> 移动端 = 重新编排的同一套亮调。原则:**单列、收紧、触摸友好、去掉一切靠悬停才成立的交互**。复杂动画的技术降级见《DEAHO_动画实现要点.md》第 8 节。

### 4.1 全局总则
- **断点** ≤767;主校准 375–430px;尊重 `env(safe-area-inset-*)`。
- **栅格** 几乎全单列;产品类 2 列(代表作/collection/合作墙可 2–3 列)。
- **间距** 段落 64–80px;容器内边距 20–24px。
- **字号** `clamp()` 下限:hero ~40–48px、h1 ~28–32px、正文 16–17px、eyebrow 11–12px。
- **触摸交互** 无 hover:zoom 改"进入即轻微放大到位"或**点击放大/lightbox**;链接酒红下划线改点击态;热区 ≥44px。
- **性能** 移动端默认关视差/Ken-Burns/漂浮/重 sticky;**触屏建议禁用 Lenis** 用原生滚动;reveal 缩短(600ms/错峰60ms);只用 transform/opacity;首屏图 `priority`,其余懒加载;遵守 prefers-reduced-motion。

### 4.2 导航(移动端)
